# 销售通 - 作战地图下钻专题 PRD (完善版)

## 1. 背景与目标
为了更好地支持业务决策，提升作战单元的拓客与管理效率，现需对“作战地图”进行大幅度优化与重构（目标Q1完成POC）。
核心聚焦于：**数据结构化展现、多层级地图无缝下钻、客户360度视图与业务模块（拜访、商机、订单等）的深度联动**。

## 2. 用户角色划分 (Data & View Permissions)
地图将根据不同角色的权限，默认展示对应层级，支持自上而下逐级下钻：
*   **CLT / 全国大总**：默认全国视角，查看各大区/战区宏观数据及对比。
*   **大区总**：默认所辖大区视角，查看下属战区、战总的绩效排行。
*   **战总 (战区总)**：默认所辖战区视角，查看具体的城市、网格及作战单元分布。
*   **作战单元 / 一线销售**：默认区域/网格视角，精细到具体客户/门店，执行日常拜访与线索跟进。

## 3. 核心业务流程与功能模块

### 3.1 地图层级与平滑下钻 (Drill-down Architecture)
基于行业领先的 GIS/地图渲染引擎（如 ECharts 3D / Deck.gl / Mapbox），实现无缝“飞入”式的下钻体验：

*   **L1：全国大区层**
    *   **视觉**：全国3D地图，以大区块（多边形）做区块热力着色（依据综合达成率）。
    *   **数据面板**：全国门店占比环图（PC品类店、乡镇专卖店等）、大区排名表、整体NetRev及达成趋势。
*   **L2：战区层**
    *   **视觉**：点击某大区（如“东北”），镜头平滑拉近，渲染该大区下的各战区图块。
    *   **数据面板**：展示对应战总（如刘克金）的考核任务($M)、实际达成($M)、综合达成率及全国/大区内排名。
*   **L3：网格/城市层**
    *   **视觉**：地图切换为散点图/聚合图 (Scatter / Clustering)，展示具体的门店点位。
    *   **数据面板**：店面数量一览、竞品趋势对比（联想 vs 华为/荣耀/小米等市场份额）。
*   **L4：客户/门店详情层 (最底层单元)**
    *   **视觉**：点击具体散点图标（红点等），弹窗或侧边栏拉出。
    *   **数据面板**：进入“客户360度超级视图”。

### 3.2 客户360度超级视图 (One-stop Customer Dashboard)
实现“不要散着做，要一环扣一环”的理念，围绕单体客户聚合所有信息：

*   **结构化标签体系**：明确客户归属与等级，显著区分 REL（消费）、ISG（基础设施）、SSG（方案服务）三大业务集团标签。
*   **企业舆情与新闻**：通过爬虫或第三方接口，实时抓取该客户的商业动态、招投标新闻、高管变动，辅助销售破冰。
*   **全链路业务追踪**：
    *   **拜访 (Visits)**：历史拜访时间轴、签到位置、会议纪要。支持直接在地图页新建拜访计划。
    *   **商机 (Opportunities)**：当前所处销售漏斗阶段、预计赢单率、预计金额。
    *   **订单 (Orders)**：历史采购产品线、合同金额、发货状态。
    *   **KPI (指标)**：针对该客户的专项拓展KPI（如提货NetRev）及距离目标的Gap。

### 3.3 报表与数据联动 (Data Table & Interactivity)
*   **双向联动**：左侧/下方的明细数据表格（如“区域提货NetRev DT考核”）需与右侧地图强绑定。
    *   *交互*：Hover 表格中的“东北-刘克金”，地图上东北战区自动高亮呼吸跳动；点击表格行，地图自动下钻至该战区。
*   **多维指标透视**：支持在地图面板顶部快捷切换筛选条件：财年(FY2526)、财季(Q4)、财周(W13)。切换后，地图着色和表格数据实现“骨架屏”无缝刷新。

## 4. 前端架构与酷炫体验设计 (Frontend Master's Plan)

*   **视觉语境 (UI/UX)**：
    *   采用 **Cyberpunk / 科技暗黑风 (Dark Mode)** 深蓝或紫黑背景色，高对比度的荧光色（青色、品红、金黄）作为数据图元的强调色。
    *   面板进场使用毛玻璃 (Glassmorphism) 过渡动画，结合悬浮流光描边效果。
*   **地图引擎选型**：
    *   建议采用 **Mapbox GL JS** （结合 Deck.gl）以支撑百万级客户节点秒级渲染，提供丝滑的 Zoom-in/Zoom-out 过渡视角。
    *   若是纯行政区划为主，也可使用 **ECharts Map 3D**，开发周期较短，契合快速 POC 验收。
*   **状态管理**：
    *   使用全局状态树管理层级路径（如 `['中国', '东北', '黑龙江']`），确保图表组件和地图组件共享最新的节点上下文，避免钻取状态不同步。

## 5. Q1 POC 验收标准 (Action Plan)
1.  **数据打通**：跑通 1~2 个真实大区（如华北、东北）的 Mock 客户数据，包含真实的组织架构（战总名字）及 KPI 目标达成。
2.  **核心交互**：演示从“全国” -> “XX大区” -> “某个门店图标”的三级丝滑下钻。
3.  **闭环卡片**：在点击特定客户点位后，能成功呼出带有“REL标签”、“一条近期新闻假数据”、“商机列表模块”的客户360面板。
