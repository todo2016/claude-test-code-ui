# 高德地图大区战区实现说明

## 1. 文档目的

本文档用于冻结 `gaode` 版本中“大区/战区”业务聚合层的设计，再在高德 JS 地图壳层上实现对应能力。

当前 `gaode` 版本已经完成以下基线：

- 使用高德标准瓦片与路网层承载底图。
- 使用 `DistrictSearch` 提供标准省、市、区行政边界。
- 复用 `v2/runtime` 中的业务数据、角色、主题与客户点逻辑。

当前缺口是：标准行政边界已经迁移到高德体系，但大区/战区仍依赖历史 GeoJSON 阶段的省份名称假设，导致业务聚合标签、区域着色、点击进入大区/战区的链路不稳定。

## 2. 目标范围

本切片只覆盖以下内容：

1. 在全国视图显示大区聚合色和大区标签。
2. 点击全国视图中的省份时，按业务归属进入对应大区视图。
3. 在大区视图显示所属战区的分色和战区标签。
4. 点击大区视图中的省份时，进入对应战区视图。
5. 在战区视图中继续承接标准省市区下钻。
6. 客户点与热力图在全国、大区、战区、行政区视图下按统一业务归属过滤。

本切片不覆盖：

- 新增真实业务数据源
- 海外区域建模
- 非省级业务聚合规则
- 新的权限模型

## 3. 业务模型

### 3.1 组织来源

组织映射继续以 `v2/runtime/map-data.js` 中的 `orgConfig` 为唯一业务基线。

- 大区：如“华东”“华南”
- 战区：如“苏皖”“京津冀”
- 省份：战区下维护省级行政区数组

`provinceToOrg` 仍然是运行时的核心查询表：

- 输入：省级行政区名称
- 输出：`{ region, zone }`

### 3.2 省份名称标准化

高德 `DistrictSearch` 返回的省级名称与历史 GeoJSON 名称可能存在以下差异：

- 带或不带“省 / 市”后缀
- 带或不带“自治区 / 特别行政区”后缀
- 业务配置使用全称，但地图返回简称

因此业务聚合层必须先做“规范省名”解析，再执行大区/战区归属判断。

标准化规则：

1. 以 `provinceToOrg` 中的省份全称作为 canonical name。
2. 对输入名做后缀剥离，生成 normalized key。
3. 用 normalized key 回查 canonical name。
4. 所有大区/战区过滤、标签锚点、颜色映射、点击跳转都基于 canonical name。

## 4. 视图行为

### 4.1 全国视图

- 基础边界：高德中国下一级省级行政区
- 着色：按省份所属大区着色
- 标签：只在每个大区的锚点省份上显示一次“大区名 + 大区”
- 点击：点击任一省份，若该省份存在业务归属，则进入对应大区视图

### 4.2 大区视图

- 基础边界：当前大区所属省份集合
- 着色：按战区分色
- 标签：每个战区只在其锚点省份上显示一次战区名称
- 点击：点击省份进入对应战区视图

### 4.3 战区视图

- 基础边界：当前战区所属省份集合
- 着色：战区单色
- 标签：显示省级行政区名称
- 点击：进入标准行政区下钻链路

### 4.4 行政区视图

- 基础边界：继续使用高德 `DistrictSearch`
- 标签：省 / 市 / 区按现有行政区逻辑显示
- 点击：按当前层级继续下钻，区级终点保留 `inlineGeoJson` 回退

## 5. 图层过滤规则

客户点与热力点的过滤规则保持一致：

1. 全国视图不过滤业务归属。
2. 大区视图按 `region` 过滤。
3. 战区视图按 `region + zone` 过滤。
4. 行政区视图按 `adcode` 或当前行政名称过滤。

当业务点的 `provinceName` 与高德边界名称不一致时，仍以 canonical province name 参与归属判断。

## 6. 降级与容错

1. 若某省份无法解析到 canonical name，则该省份不参与大区/战区聚合，但保留标准行政边界展示。
2. 若某大区锚点省份缺失，则该大区不显示聚合标签，但点击其成员省份仍可进入大区视图。
3. 若高德 `DistrictSearch` 失败，则继续使用当前边界服务中的 GeoJSON fallback。

## 7. 实现切片

1. 在 `gaode/runtime/map-shell.js` 中新增省份 canonical name 解析能力。
2. 替换大区/战区相关的所有省份名称匹配逻辑。
3. 保持 `v2/runtime/map-data.js` 作为业务组织配置源，不复制第二份组织映射。
4. 不改变当前 `drilldown-state.js` 的状态模型，只修正进入这些状态所依赖的业务匹配逻辑。
